About: This page presents a quarterly overview of the planned features and enhancements to AMP. Projects are organized according to core focus areas.

This document will be updated twice per quarter:

  • The mid-quarter update will provide the first listing of the current quarter’s new priorities and present status of projects. In addition, a forecast of priorities for the following two quarters will be provided.
  • The end-of-quarter update will present the status of the just-ended quarter’s priorities and projects, as well as any updates to the two-quarter lookahead. Priorities for the quarter just being entered will be presented at the next mid-quarter update.

Stay up to date: You can track regular releases of the AMP Project at our Releases page on GitHub or read news at our blog. To subscribe to announcements, please join our mailing list: amphtml-announce@googlegroups.com.

Focus Area

2016/Q2

2016/Q3 & 2016/Q4 (Forecast)

Format

Themes

  • Enable new types of pages to be supported in AMP, such as automatically updating liveblogs
  • Broaden AMP’s feature support for relevance in retail and travel content verticals
  • Make it easier to build well-styled and interactive AMP pages
  • Provide “full site” behaviors like navigation menus and search boxes
  • Support data entry use cases (e.g. newsletter signups, product customization)
  • Further alignment with Progressive Web Apps (PWAs)

Stage 6: Launched / Stable

  • Cross-origin support for ServiceWorker installation [update to amp-install-serviceworker ]
  • Site navigation menu support [ amp-sidebar ]

Stage 5: Code Complete / Experimental

  • Share button bar  [ amp-social-share ]

Stage 4: Pull Request (Changelist) Created

  • Live blog support[a][b][c]
  • Form support

Stage 1: In Planning and Design

  • Native app install button/banner
  • “AMP bootstrap” default styling
  • amp-carousel improvements
  • amp-lightbox improvements
  • Geo-variation support
  • Create a suite of effects that can be applied to AMP page content and structure
  • More form support, including validation, conditional behaviors, grouping
  • Introduce a mechanism by which element behavior can be bound to user actions
  • Offline support

Analytics

Themes

  • Support element-level analytics better
  • Support A/B experience testing
  • Support innovative/interesting types of analytics

Stage 4: Pull Request (Changelist) Created

  • Element-level visibility trigger support in amp-analytics

Stage 2: Intent to Implement (Feature Description) Posted

  • Share-tracking support in amp-analytics
  • A/B experience testing support in amp-analytics

Stage 0: Not Started

  • Element-level data variable support in amp-analytics
  • Expose events from video players to amp-analytics
  • Expose batched interaction data (e.g. scroll events) to amp-analytics
  • Expansion of analytics capabilities for needs in more verticals, such as ecommerce

Ads

Themes

  • Enable new types of ad formats that can be used in AMP pages
  • Enable AMP itself as an ad creative format
  • Enable AMPs to act as landing pages for ads

Stage 4: Pull Request (Changelist) Created[d][e][f][g][h][i][j]

  • Sticky ad format in amp-ad
  • Flying carpet effect support, particularly as ad format

Stage 3: [k][l][m][n] Coding Started

  • Allow for serving ads that are in AMP format  (A4A - AMP for Ads )

Stage 1: In Planning and Design

  • Pre-roll video ads in amp-video using IMA
  • Proof of concept implementation of ads with AMP landing pages
  • Enhancements to AMP runtime to allow passing AMP signals to ad servers and ad exchange[o][p][q] s
  • Enabling “AMP for Ads” creative demand across the buyer and publisher ecosystem
  • Enabling deeper integration of Ads with AMP landing pages in the various buyside and sellside systems

Access

Themes

  • Continue to improve paywall and subscription support in AMP
  • Make sign-in and payment for content a seamless and joyful experience in AMP

Stage 5 : Code Complete/Experimental

  • AMP-Access server side support (limited beta)[r][s][t]

Stage 1: In Planning and Design

  • Runtime support for delegated sign-in (allow surface hosting AMP to sign-in user to publisher from using available identity information)
  • Enabling better mobile payments with AMP
